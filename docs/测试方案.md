# PulseBrief 测试方案

## 测试目标

验证首个可运行 MVP 是否覆盖产品设计文档中的核心体验，并为后续 Spring Boot、Flutter、MySQL 接入保留稳定回归基线。

## 自动化测试

### 业务逻辑测试

覆盖范围：

1. 首页排序：置顶、突发、订阅分类、热度、发布时间的优先级。
2. 分类筛选：按分类编码返回对应资讯。
3. 订阅规则：默认订阅 3 个分类，少于 3 个不允许完成。
4. 收藏状态：收藏和取消收藏能正确切换。
5. 播报队列：单条资讯和简报能生成当前播放项。

### 组件烟测

覆盖范围：

1. 首页能展示今日热点、订阅 Tab 和资讯列表。
2. 分类页能切换分类并展示当前分类说明。
3. 简报页能展示早报、午间快讯、晚间复盘。
4. 我的页面能展示订阅、收藏、播放设置和推送设置。
5. 后台页能展示运营指标和管理列表。

## 构建验证

每次交付前执行：

```bash
npm install
npm test
npm run build
```

预期结果：

1. 所有测试通过。
2. TypeScript 无类型错误。
3. Vite build 成功生成 `apps/web/dist`。

## 手工回归清单

用户端：

1. 打开应用后进入登录/兴趣选择流程。
2. 默认选中“全球热点、财经市场、科技趋势”。
3. 少于 3 个订阅时完成按钮不可用。
4. 完成后进入首页。
5. 首页点击资讯卡片进入详情页。
6. 详情页可收藏、播放、返回。
7. 分类页点击不同分类后列表变化。
8. 简报页点击播放后底部播放器出现。
9. 我的页面可查看收藏和播放设置。

后台端：

1. 从顶部入口切换到后台。
2. 数据看板展示核心指标。
3. 资讯源、文章、简报、推送模块有可读列表。
4. 返回用户端后底部导航正常。
