# PulseBrief 测试方案

## 测试目标

验证正式 V1 工程主线是否保持可构建、可测试，并为后续 Spring Boot、Flutter、MySQL 联调保留稳定回归基线。

## 自动化测试

### 后端测试

覆盖范围：

1. Spring Boot 应用入口存在且可调用。
2. `/api/health` 返回服务状态。
3. 后续补充分类、文章、简报、收藏、订阅、播放历史测试。

### Flutter 测试

覆盖范围：

1. 当前默认 widget 测试保持通过。
2. 后续补充首页、分类、详情、每日简报、收藏、播放器入口的 widget 测试。
3. UI 稳定后补充 golden 测试。

## 构建验证

每次交付前执行：

```powershell
.\scripts\check-env.ps1

.\scripts\use-jdk17.ps1
cd backend
.\mvnw.cmd test

cd ..\mobile
.\..\scripts\use-flutter.ps1
flutter test

cd ..
docker compose -f deploy/docker-compose.yml config
```

预期结果：

1. 所有测试通过。
2. 后端 Spring Boot 测试通过。
3. Flutter widget 测试通过。
4. Docker Compose 配置有效。

## 手工回归清单

用户端：

1. Flutter App 可启动。
2. 下一阶段 UI 实现后，回归登录/兴趣选择、首页、分类、详情、简报、我的页面。
3. 收藏、播放、订阅等交互在 Flutter 内完成验证。

后台端：

1. 正式 React Admin 创建后再补充。
2. 后台不复用已删除的旧 PWA。
