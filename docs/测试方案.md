# PulseBrief 测试方案

## 测试目标

验证正式 V1 工程主线是否保持可构建、可测试，并为后续 Spring Boot、Flutter、MySQL 联调保留稳定回归基线。

## 自动化测试

### 后端测试

覆盖范围：

1. Spring Boot 应用入口存在且可调用。
2. `/api/health` 返回服务状态。
3. 分类、文章、简报、收藏、订阅、播放历史测试按 V1 接口分批补充。

V1 核心接口测试：

1. 分类：`GET /api/categories` 返回启用分类并按 `sortNo` 排序。
2. 首页：`GET /api/articles/home` 返回今日简报、投行观点推荐和资讯列表。
3. 文章：`GET /api/articles` 支持分类分页；`GET /api/articles/{id}` 返回结构化摘要且不返回新闻全文。
4. 简报：`GET /api/digests/today` 和 `GET /api/digests/{id}` 返回播报所需字段。
5. 登录：`POST /api/auth/login` 校验协议同意状态；`POST /api/auth/guest` 支持游客模式。
6. 订阅：未登录返回 401；登录后可查询和保存订阅。
7. 收藏：收藏与取消收藏保持幂等。
8. 播放历史：文章和简报播放记录均可写入。

数据库回归：

1. Flyway migration 在空库可初始化。
2. 种子数据包含分类、来源、文章和简报。
3. `news_article.article_hash` 唯一约束生效。

### Flutter 测试

覆盖范围：

1. 当前默认 widget 测试保持通过。
2. 后续补充首页、分类、详情、每日简报、收藏、播放器入口的 widget 测试。
3. UI 稳定后补充 golden 测试。

```powershell
.\mvnw.cmd test

```

预期结果：

1. 所有测试通过。

4. Docker Compose 配置有效。

### Flutter 高保真 UI 精修回归

精修阶段以 Android 模拟器为主要视觉验收环境，Flutter Web 移动宽度预览作为辅助检查。

页面检查：

1. 登录页：品牌区、输入框、验证码按钮、主按钮、游客入口、协议勾选与原型接近。
2. 首页：顶部问候、今日全球简报主卡、分类 Tab、投行观点卡、热点资讯列表、底部导航不遮挡。
3. 分类页：横向分类导航、当前分类卡、今日重点双列卡、分类资讯列表与原型接近。
4. 订阅页：概览卡、推荐订阅网格、重点频道、推送偏好、底部保存区状态正常。
5. 详情页：AI 摘要、核心要点、可能影响、相关资讯、底部操作栏不溢出。
6. 简报页：今日早报主卡、简报列表、今日热点清单、底部导航高亮正确。
7. 播放器页：封面卡、进度条、播放控制、倍速卡、当前播报要点布局正常。
8. 我的页：头像昵称区、会员卡、菜单分组、简报偏好、底部协议链接和导航安全区正常。

交互检查：

1. 登录 / 游客模式进入首页。
2. 首页、分类、简报、我的底部导航切换。
3. 资讯卡片进入详情页。
4. 播放入口进入播放器页。
5. 收藏、播放暂停、倍速、分类、订阅标签和推送开关均可切换。
