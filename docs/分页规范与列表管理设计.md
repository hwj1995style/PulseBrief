# PulseBrief 分页规范与列表管理设计

## 背景与现状

V1 用户中心列表已完成收藏、阅读历史和播放历史三个闭环，但接口仍直接返回数组，缺少统一分页元信息；Flutter 列表页也只支持一次性加载，用户无法在列表内完成常见管理动作。下一阶段需要在不扩大复杂度的前提下补齐 V1.1 列表规范和最小管理能力。

## 目标与非目标

目标：

1. 新增统一分页响应 `PageResponse<T>`。
2. `GET /api/user/favorites`、`GET /api/user/read-history`、`GET /api/playback/history` 返回分页结构。
3. 收藏列表支持单条取消收藏。
4. 阅读历史支持清空。
5. 播放历史支持清空。
6. Flutter API Repository 能兼容分页响应。
7. Flutter 列表页支持刷新和基础管理动作。

非目标：

1. 不做复杂无限滚动虚拟列表。
2. 不做批量多选管理。
3. 不做单条阅读历史或播放历史删除。
4. 不引入新的第三方状态管理库。
5. 不改变首页、分类页等现有数组接口。

## 影响范围

后端：

1. 新增 `common.api.PageResponse`。
2. 修改收藏、阅读历史、播放历史列表接口响应类型。
3. 新增 `DELETE /api/user/read-history` 清空阅读历史。
4. 新增 `DELETE /api/playback/history` 清空播放历史。
5. Repository 增加用户维度删除方法。

Flutter：

1. `ApiPulseRepository` 支持从分页响应中读取 `items`。
2. `PulseRepository` 增加 `clearReadHistory()`、`clearPlaybackHistory()`。
3. `UserArticleListPage` 支持刷新；收藏列表的收藏按钮可取消收藏并从列表移除。
4. `PlaybackHistoryPage` 支持刷新和清空。

## API 设计

分页响应：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 42,
  "hasMore": true
}
```

受影响接口：

```text
GET /api/user/favorites
GET /api/user/read-history
GET /api/playback/history
```

新增接口：

```text
DELETE /api/user/read-history
DELETE /api/playback/history
```

清空接口响应：

```json
{
  "cleared": true
}
```

## 数据模型与权限模型

1. 分页元信息由对应用户维度表 `countByUserId` 生成。
2. 清空阅读历史删除当前用户的 `user_read_history` 记录。
3. 清空播放历史删除当前用户的 `user_play_history` 记录。
4. 所有列表和清空接口都要求开发态 token。

## 后端实现方案

1. 新增 `PageResponse.of(items, page, pageSize, total)`。
2. Service 方法保留 `page/pageSize` 入参，返回 `PageResponse`。
3. Repository 增加 `deleteByUserId`。
4. Controller 测试先切换到 `data.items` 和分页字段。

## 前端影响

1. Flutter `_dataList` 增强为兼容：
   - `data` 是数组：按旧逻辑读取。
   - `data` 是对象且包含 `items`：读取 `items`。
2. 收藏列表点击已收藏图标时调用 `unfavoriteArticle` 并移除该条。
3. 阅读历史页面头部提供清空按钮。
4. 播放历史页面头部提供清空按钮。
5. 刷新使用 `RefreshIndicator` 包裹当前列表。

## 测试与回归方案

后端：

1. 收藏列表测试断言 `data.items[0]` 和 `data.total`。
2. 阅读历史列表测试断言分页结构。
3. 播放历史列表测试断言分页结构。
4. 阅读历史清空测试断言 `cleared=true`。
5. 播放历史清空测试断言 `cleared=true`。

Flutter：

1. Repository 测试使用分页响应 fixture。
2. Repository 测试覆盖清空阅读历史和播放历史。
3. Widget 测试覆盖收藏列表取消收藏按钮存在、历史页清空按钮存在。
4. live API 测试覆盖分页读取和清空接口。

## 风险与分阶段落地

1. 返回结构从数组变为分页对象会影响旧客户端；Flutter 本轮做兼容读取，后续可统一所有列表。
2. 清空历史是不可逆动作，V1.1 暂不做二次确认弹窗自动化测试，页面使用清晰图标入口。
3. 分页只先规范响应，不在本轮实现无限滚动。
