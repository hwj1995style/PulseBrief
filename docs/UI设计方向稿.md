# PulseBrief Flutter UI 设计方向稿

## 背景与现状

PulseBrief 已移除早期 React/Vite PWA MVP。正式用户端将以 Flutter 实现，旧 PWA 的视觉样式不再作为参考。

本次 UI 设计使用 Figma 生成可编辑方向稿，目标是确定 V1 用户端的主流移动资讯 App 风格，再进入 Flutter 实现。

Figma 文件：

https://www.figma.com/design/9EfobQfJgjSooTIPJ2tobU

## 设计目标

1. 建立 PulseBrief 正式移动端视觉方向。
2. 覆盖首页、分类、资讯详情、每日简报四个核心页面。
3. 强化 AI 摘要、每日简报、语音播报的产品识别。
4. 保持 Flutter 可落地，避免复杂动效、重图片和过度装饰。
5. 形成后续颜色、字体、间距、组件和页面规范的基础。

## 非目标

1. 本次不实现 Flutter 代码。
2. 本次不设计 React Admin。
3. 本次不复用旧 PWA 视觉。
4. 本次不引入新闻全文展示或版权风险较高的内容模块。

## 设计方向

### 方向 1：Material Brief

定位：

主流 Material 3 移动资讯体验，清爽、稳重、跨 iOS / Android 易落地。

视觉特征：

1. 浅灰绿色背景和白色内容面。
2. 绿色主色用于简报、播放、订阅标签和关键操作。
3. 首页顶部强化“今日简报”和一键播放。
4. 列表信息密度适中，以轻分隔线和标签承载层级。
5. 底部迷你播放器常驻，突出语音播报能力。

适用判断：

推荐作为 Flutter V1 默认方向。它最适合跨平台实现，审美主流，复杂度可控。

### 方向 2：Apple Editorial

定位：

偏 Apple News 的精品编辑感，强调阅读质感和每日精选。

视觉特征：

1. 白底、大标题、克制分隔线。
2. 黑白灰为主，少量玫红色作为品牌强调。
3. 信息流更像编辑精选，而不是传统门户列表。
4. 详情页阅读感更强，适合高质量摘要和结构化解读。

适用判断：

适合强调“每日精品简报”的产品气质，但 Android 端落地时需要额外控制平台一致性。

### 方向 3：Pro Market Brief

定位：

更专业、更密集的财经 / 投行信息面板，面向投资者和高信息密度用户。

视觉特征：

1. 深色背景和深色内容面。
2. 蓝色用于信息强调，黄色用于播放和重点提示。
3. 列表更密集，更接近市场观察工具。
4. 适合财经市场、投行观点、宏观政策等场景。

适用判断：

适合专业用户，但首版大众化门槛更高，可作为后续深色模式或专业模式方向。

## 推荐结论

建议选择方向 1：Material Brief 作为 PulseBrief V1 默认 UI。

原因：

1. 和 Flutter / Material 体系天然匹配。
2. iOS 和 Android 的一致性最好控制。
3. 能承载资讯流、AI 摘要、每日简报、播放器等全部核心模块。
4. 主流审美足够稳，不会像旧 PWA 一样显得网页化。
5. 后续可以平滑扩展深色模式和专业财经模式。

## 页面范围

Figma 方向稿已覆盖：

1. 首页：日期、品牌、搜索、今日简报、订阅分类、热点列表、迷你播放器、底部导航。
2. 分类页：分类标签、当前分类重点、资讯列表、底部导航。
3. 资讯详情页：标题、来源、AI 摘要、核心要点、可能影响、收藏、分享、播放入口。
4. 每日简报页：早报卡片、播放入口、简报分段、迷你播放器、底部导航。

后续最终 UI 需要补齐：

1. 启动页。
2. 登录页。
3. 兴趣选择页。
4. 我的页面。
5. 订阅管理。
6. 收藏列表。
7. 播放历史。
8. 完整播放器。
9. 推送设置。
10. 异常、空状态、加载状态。

## Flutter 落地约束

1. 使用 Material 3 ThemeData 建立主题。
2. 使用统一颜色 Token，不在页面里散落硬编码颜色。
3. 页面组件先拆为 `NewsCard`、`CategoryChip`、`DailyBriefCard`、`MiniPlayer`、`BottomNavShell`。
4. 首版优先完成静态 mock 数据交互，再接后端 API。
5. 首页、分类、详情、简报的 widget 测试随实现同步补充。
6. 迷你播放器先做前台 UI 状态，TTS 后续接入。
