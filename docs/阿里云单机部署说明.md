# PulseBrief 阿里云单机部署说明

当前部署面向个人自用，不接入应用商店、FCM 或 APNs。生产拓扑为 Caddy、React Admin、Spring Boot、MySQL 四个容器，只有 Caddy 的 80/443 端口暴露到公网。

## 本机准备

1. 域名 `huawj.com.cn` 的 A/AAAA 或代理源站指向 `47.112.14.22`。
2. 阿里云安全组放行 TCP 80、443，并仅向可信来源开放 TCP 22。
3. SSH 私钥保存在 `.local-secrets/server/aliyun1.pem`，生产配置与初始管理员凭据保存在同一 Git 忽略目录。
4. `.env.local` 中已配置重新生成的 DeepSeek Key。

首次初始化：

```powershell
.\scripts\initialize-production-secrets.ps1
```

生产发布仅允许从干净的 `main` 执行：

```powershell
.\scripts\deploy-aliyun.ps1
```

脚本会校验私钥指纹、上传当前 `main` 的 Git 归档、补充 2GB Swap、配置防火墙、构建并启动容器、验证健康接口和管理员登录。首次登录验证通过后，服务器和本机运行环境会移除管理员引导用户名与密码，初始登录密码只保留在 `.local-secrets/server/admin-initial-credentials.txt`，首次登录后应立即修改。

## 服务端目录

- `/opt/pulsebrief/repo`：当前发布版本软链接。
- `/opt/pulsebrief/releases`：按 UTC 时间保存的发布版本，可用于人工回滚。
- `/opt/pulsebrief/.env`：权限为 600 的运行 Secret。
- `/var/backups/pulsebrief`：MySQL 每日压缩备份，默认保留 14 天。

## 常用检查

```bash
cd /opt/pulsebrief/repo/deploy/production
docker compose --env-file /opt/pulsebrief/.env ps
docker compose --env-file /opt/pulsebrief/.env logs --tail=200 backend gateway
systemctl status pulsebrief-backup.timer
```

公开健康地址为 `https://huawj.com.cn/api/health`，Admin 地址为 `https://huawj.com.cn`。Swagger 与 OpenAPI 文档在生产网关被禁止访问。

## 备份恢复边界

定时任务每天 02:30（服务器本地时区）备份 MySQL，并生成 SHA-256。恢复属于有状态变更，执行前必须停止写入、另行备份当前数据并人工确认目标文件；部署脚本不会自动覆盖数据库。
