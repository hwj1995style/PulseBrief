# PulseBrief 用户历史列表接口设计

## 背景与现状

用户中心资料接口已完成，“我的”页可以展示订阅数、收藏数、阅读数和播放数。但“我的收藏”“阅读历史”“播放历史”三个入口仍没有真实列表页面，后端也只有收藏写入、播放历史写入能力，缺少用户侧列表查询。`user_read_history` 表已在 V1 初始化脚本中存在，但当前文章详情打开时尚未写入阅读记录。

## 目标与非目标

目标：

1. 新增用户收藏列表接口，支持“我的收藏”页面。
2. 新增用户阅读历史列表接口，并在文章详情打开时记录阅读历史。
3. 新增播放历史列表接口，支持“播放历史”页面。
4. Flutter Repository 增加对应读取方法。
5. Flutter “我的”页三个入口可跳转到真实列表页。
6. mock 模式继续可独立运行。

非目标：

1. 不做复杂分页组件，V1 先用 `page` 和 `pageSize` 返回数组。
2. 不做批量删除、清空历史或收藏管理。
3. 不实现真实用户体系或生产 JWT。
4. 不新增全文阅读页，详情页仍只展示结构化摘要。

## 影响范围

后端：

1. `favorite`：新增收藏列表查询。
2. `playback`：新增播放历史列表查询。
3. `readhistory`：新增阅读历史实体、Repository、Controller、Service。
4. `article`：文章详情支持可选 Authorization，登录态下记录阅读历史。
5. `user profile`：`readCount` 改为读取 `user_read_history`。

Flutter：

1. `PulseRepository` 增加收藏、阅读历史、播放历史列表方法。
2. `ApiPulseRepository` 接入新增接口。
3. `MockPulseRepository` 返回 mock 列表。
4. 新增用户列表页：收藏列表、阅读历史、播放历史。
5. “我的”页入口增加跳转。

## API 设计

### GET /api/user/favorites

用途：查询我的收藏，需要登录。

查询参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `page` | `1` | 页码 |
| `pageSize` | `20` | 每页数量，最大 50 |

响应：文章卡片数组。

### GET /api/user/read-history

用途：查询阅读历史，需要登录。

查询参数同收藏列表。响应为文章卡片数组，按最近阅读时间倒序。

### POST /api/user/read-history

用途：记录阅读历史，需要登录。

请求：

```json
{
  "articleId": 1
}
```

响应：

```json
{
  "id": 100
}
```

### GET /api/playback/history

用途：查询播放历史，需要登录。

查询参数同收藏列表。响应：

```json
[
  {
    "id": 1,
    "playType": "ARTICLE",
    "articleId": 1,
    "digestId": null,
    "playTitle": "高盛：AI 基建投资仍将持续",
    "playTime": "2026-06-09T09:30:00+08:00",
    "durationSeconds": 168
  }
]
```

## 数据模型与权限模型

1. `user_favorite`：通过 `user_id` 查询用户收藏，再关联 `news_article`。
2. `user_read_history`：每次文章详情打开写入一条记录，V1 暂不去重。
3. `user_play_history`：播放写入已存在，本轮补列表查询。
4. 所有用户列表接口都要求 `Authorization: Bearer dev-token-1`。
5. 文章详情接口允许无登录访问；若带合法 token，则异步等价地记录阅读历史。V1 先同步写入，失败不影响详情返回。

1. 复用 `ArticleCardResponse` 作为收藏和阅读历史列表项。

1. 新增 `PlaybackHistoryItem` model。
2. 新增 `UserArticleListPage` 展示收藏和阅读历史文章卡片。

## 风险与分阶段落地

1. 阅读历史 V1 暂不去重，统计可能随多次打开详情快速增长；后续可改为按用户文章 upsert 并更新时间。
2. 列表 V1 返回数组，后续数据变多时再引入统一分页响应。
3. 播放历史可能包含简报和文章两种类型，Flutter 首轮以通用历史条目展示，后续再根据类型跳转详情。
