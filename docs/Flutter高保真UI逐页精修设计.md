# PulseBrief Flutter 高保真 UI 逐页精修设计

## 背景与现状

PulseBrief 当前 Flutter 移动端已完成登录页、首页、分类、订阅、资讯详情、每日简报、语音播放器、我的页面的基础高保真实现，并已能在 Android 模拟器中运行。现有实现具备页面结构、路由、mock 数据和基础交互，但与用户提供的高保真原型相比，仍偏“结构可用版”，在真实 APP 质感、卡片密度、视觉层级、底部导航、图文比例和页面节奏上需要逐页精修。

本次精修以用户提供的 8 张原型图为视觉来源：

1. 登录页。
2. 首页。
3. 分类页。
4. 订阅页。
5. 资讯详情页。
6. 每日简报页。
7. 语音播放器页。
8. 我的页面。

## 目标与非目标

目标：

1. 对比原型逐页精修 Flutter 原生页面，使移动端效果更接近真实 iOS 风格资讯 APP。
2. 保持深蓝主色、浅灰白背景、少量青绿色点缀的专业财经科技风格。
3. 优先修正公共视觉基线：页面边距、卡片圆角、阴影、描边、字体层级、按钮高度、底部安全区和导航栏形态。
4. 逐页对齐核心布局：顶部标题区、主视觉卡片、列表卡片、操作区和底部导航。
5. 保持 mock 数据和现有路由，继续预留后续 Spring Boot API 接入边界。
6. 保证 Android 模拟器和 Flutter Web 移动宽度预览均可用，但验收以 Android 模拟器为主。

非目标：

1. 不接入真实后端接口。
2. 不实现真实登录、短信验证码、推送、TTS、会员支付或研报下载。
3. 不新增 H5 或 WebView 套壳。
4. 不引入复杂第三方 UI 框架。
5. 不重做信息架构，不改变当前 8 个页面和主要路由。
6. 不追求像素级 100% 复制状态栏时间、电量等系统区域。

## 后端实现方案

本次不修改后端。

后续后端接入建议：

1. 首页、分类、详情、简报、订阅和用户偏好分别定义 API DTO。
2. Flutter 侧新增 repository 层，将 mock 数据替换为异步数据源。
3. 页面保留 loading、empty、error 状态组件，便于接入真实网络状态。

## 前端精修方案

### 公共视觉基线

先统一公共层，再逐页落地：

1. `AppColors`：补充更接近原型的背景、蓝色渐变、浅蓝描边、柔和阴影色和成功青色。
2. `AppTextStyles`：细化超大标题、页面标题、卡片标题、正文、meta、按钮文字；移动端中文字号避免过大导致换行失控。
3. `AppSpacing`：补充页面顶部、卡片内边距、列表间距、底部导航预留高度。
4. `AppRadius`：统一输入框、卡片、图片、按钮和底部导航的圆角层级。
5. `AppShadows`：区分卡片轻阴影、导航强阴影、主按钮阴影。

### 共享组件

重点精修以下组件：

1. `AppHeader`：适配原型中的大标题、居中标题、返回按钮和右侧操作按钮。
2. `PulseCard`：支持轻描边、浅蓝背景、柔和阴影、不同 padding。
3. `BriefHeroCard`：增强图文比例、右侧图片裁切、主按钮和次按钮组合，使首页、分类、简报、播放器封面更贴近原型。
4. `NewsCard`：拆分横向资讯卡、紧凑重点卡和双列重点卡，避免所有列表都长得一样。
5. `CategoryChip`：调整高度、圆角、边框和选中态，订阅页支持带勾选图标。
6. `PulseBottomNav`：做成原型中的 iOS 浮层底栏，中间简报按钮凸起并保持安全区。
7. `MiniPlayer`：如保留在首页底部，应避免遮挡列表，并接近原型的悬浮播报条质感。

## 逐页精修方案

### 1. LoginPage

目标：

1. 顶部品牌区域更接近原型：视觉居中、背景图上移、Logo 与“脉闻 / PulseBrief”比例更稳。
2. 登录卡片控制宽度和圆角，避免 Web 宽屏时拉满。
3. 输入框高度、图标、分割线、验证码按钮和主按钮高度贴近原型。
4. 游客模式和稍后登录按钮保持大圆角描边卡片。
5. 协议勾选区域在小屏不溢出。

验收点：

1. Android 模拟器截图接近原型，不再出现桌面浏览器宽屏拉伸问题。
2. 未勾选协议时点击登录提示用户先同意协议。
3. 登录和游客入口仍能进入首页。

### 2. HomePage

目标：

1. 顶部标题区恢复原型中的日期、问候语、产品名和右侧通知 / 搜索入口节奏。
2. 今日全球简报主卡片增强宽度、图文比例、按钮位置和浅蓝科技图背景。
3. 分类快捷 Tab 的尺寸、间距和选中态贴近原型。
4. 投行观点推荐卡做成浅蓝强调卡。
5. 热点资讯卡片按原型改为更紧凑的横向图文列表，保留播放与收藏入口。
6. 底部 mini player 和底部导航不互相遮挡。

验收点：

1. 首页首屏能同时看到主卡片、分类 Tab 和第一条资讯，不显得像后台或网页。
2. 点击资讯进入详情，点击播放进入播放器。
3. 收藏状态可切换。

### 3. CategoryPage

目标：

1. 对齐原型分类页：顶部“分类”标题、说明文字、搜索按钮和横向分类导航。
2. 当前分类说明卡片使用浅蓝背景和右侧财经/全球图。
3. 今日重点模块支持双列重点卡，小屏自动单列。
4. 分类资讯列表更接近原型的卡片密度和图片比例。

验收点：

1. 点击分类可切换选中状态和 mock 内容。
2. 分类页底部导航高亮分类。
3. 小屏不出现双列卡片溢出。

### 4. SubscriptionPage

目标：

1. 顶部返回、居中标题、副标题和保存按钮贴近原型。
2. 已订阅概览卡片强化右侧插图、选中标签和今日筛选提示。
3. 推荐订阅标签使用三列网格，选中态带勾。
4. 重点频道卡片为两列，图标、标题、说明、开关对齐原型。
5. 推送偏好使用 iOS 风格列表与开关。
6. 底部保存按钮固定但不遮挡内容。

验收点：

1. 标签和开关均可切换。
2. 点击保存显示成功提示。
3. 底部导航高亮我的。

### 5. ArticleDetailPage

目标：

1. 顶部返回和分享按钮间距贴近原型。
2. 标题字号、行高和 meta 区域更像原型中的详情页。
3. AI 摘要、核心要点、可能影响、相关资讯四个模块统一为结构化卡片。
4. 底部操作栏优化为原型中的四项操作和主按钮，支持安全区。

验收点：

1. 不展示新闻全文。
2. 语音播报可进入播放器。
3. 收藏状态可切换。

### 6. DigestPage

目标：

1. 顶部标题、日期、说明文字和日历按钮贴近原型。
2. 今日全球早报主卡片突出语音简报能力，按钮组合为播放整篇和查看全文。
3. 简报分类列表使用更明确的图标色块、更新时间和播放按钮。
4. 今日热点清单按原型双列排名布局，小屏单列。

验收点：

1. 简报页看起来不是普通文章列表。
2. 播放整篇和单条播放可进入播放器。
3. 底部导航高亮简报。

### 7. PlayerPage

目标：

1. 顶部返回、居中标题、播放列表按钮贴近原型。
2. 封面卡片图文比例更接近原型。
3. 标题、来源、更新时间、总时长、进度条和时间文本对齐。
4. 上一条、播放 / 暂停、下一条按钮尺寸和间距贴近原型。
5. 倍速、定时关闭、播放列表、跳转原文、收藏控制区改为一张操作卡。
6. 当前播报要点列表更紧凑、编号更清晰。

验收点：

1. 播放 / 暂停状态可切换。
2. 倍速可切换。
3. 收藏状态可切换。

### 8. MinePage

目标：

1. 顶部“我的”、通知、设置按钮贴近原型。
2. 头像、昵称、认证标识和简介排版更接近原型。
3. 会员卡、常用入口卡、设置卡、简报偏好卡分组清晰。
4. 菜单行的图标、间距、右侧说明和分割线统一。
5. 底部协议链接不拥挤，不与导航栏冲突。

验收点：

1. 点击我的订阅进入订阅页。
2. 页面分组清晰，小屏不拥挤。
3. 底部导航高亮我的。

## 风险与分阶段落地建议

风险：

1. 原型图是静态图片，部分图片素材和状态栏细节无法完全复刻。
2. 当前素材为本地生成/裁切图，正式上线前需要替换为版权清晰的真实内容图或自有资产。
3. 一次性精修 8 个页面容易造成样式漂移，需要先改公共组件再逐页验收。
4. Flutter Web 预览和 Android 模拟器渲染存在差异，最终以模拟器为准。

分阶段建议：

1. 第一批：公共 theme、公共组件、登录页、底部导航。
2. 第二批：首页、分类页、资讯卡片体系。
3. 第三批：简报页、播放器页、详情页。
4. 第四批：订阅页、我的页、整体截图回归。
5. 第五批：补 widget 测试和必要的截图验收记录。
